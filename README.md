## ENGINEERING PRINCIPLES

</div>

<table width="100%">
<tr>
<td width="33%" align="center" valign="top">

### Correctness First

Code that works is the baseline. Code that works under edge cases, handles errors gracefully, and fails predictably is engineering. Write tests. Handle errors. Document assumptions.

</td>
<td width="33%" align="center" valign="top">

### Performance Matters

Latency compounds. Memory leaks crash servers. N+1 queries kill databases. Profile before optimizing, but always be aware of complexity. Fast code is useless if it's wrong, but slow code is useless too.

</td>
<td width="33%" align="center" valign="top">

### Clarity Over Cleverness

The best code is boring. Predictable patterns, clear names, obvious structure. Future engineers will thank you. Your future self will thank you. Clever code is a liability.

</td>
</tr>
</table>
